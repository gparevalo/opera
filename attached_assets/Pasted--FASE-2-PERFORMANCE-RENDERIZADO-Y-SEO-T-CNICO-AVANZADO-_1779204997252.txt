# AUDITORÍA DE PERFORMANCE

Analizar:

* Largest Contentful Paint (LCP)
* First Input Delay (FID/INP)
* Cumulative Layout Shift (CLS)
* Time To First Byte (TTFB)
* render blocking
* hydration cost
* bundle size
* unused JavaScript
* unused CSS
* layout shifts
* oversized assets

Detectar:

* cuellos de botella,
* componentes pesados,
* problemas de renderizado,
* animaciones costosas,
* exceso de re-renders.

---

# SSR Y RENDERIZADO — CRÍTICO

Validar:

* qué contenido se renderiza server-side,
* qué contenido depende completamente de JS,
* qué contenido no puede leer un crawler,
* qué secciones podrían perder indexación.

Priorizar:

* SSR,
* contenido HTML visible,
* semantic rendering,
* indexabilidad real.

Evitar:

* páginas vacías para crawlers,
* renderizado exclusivamente client-side,
* contenido importante oculto en animaciones.

---

# SCROLL Y ANIMACIONES — MUY IMPORTANTE

El sitio tiene:

* motion,

* scrollytelling.

Optimizar para que:

* no rompan SEO,

* no oculten contenido crítico,
* no generen CLS.

Mantener:

* fallback semántico,

* accesibilidad del contenido.
